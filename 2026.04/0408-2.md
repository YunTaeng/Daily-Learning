## 💡 학습 내용 (Main Concepts)

### 1. Static 멤버 vs 인스턴스 멤버

* **Static (정적) 멤버:** 클래스 로딩 시점에 생성되어 객체 생성 없이 클래스 이름으로 즉시 접근 가능하다.
* **인스턴스 멤버:** 반드시 `new` 연산자로 객체를 생성해야만 힙(Heap) 메모리에 할당되어 사용할 수 있다.
* **핵심:** `static` 메서드(main) 내부에서는 메모리에 아직 존재하지 않는 인스턴스 필드나 메서드에 직접 접근할 수 없다.

### 2. 메서드 설계와 오버로딩 (Overloading)

* **Return 문:** 메서드 실행을 종료하고 결과값을 호출한 곳(Caller)으로 전달한다. `void`는 반환값이 없을 때 사용한다.
* **메서드 오버로딩:** 이름은 같지만 **파라미터의 개수나 타입**이 다른 메서드를 중복 선언하는 것이다.
* **이점:** 동일한 기능을 다양한 타입의 데이터로 처리할 때 가독성과 편의성을 높여준다.

### 3. 생성자 (Constructor)와 초기화

* **역할:** 객체 생성 시 필드를 초기화하거나 필수 로직을 실행한다.
* **특징:** 클래스 이름과 동일해야 하며 리턴 타입이 없다. 생성자를 정의하지 않으면 컴파일러가 기본 생성자를 자동으로 추가한다.

### 4. 참조 키워드: `this` vs `super`
두 키워드 모두 메모리상의 객체를 가리키는 참조 변수이지만 용도가 다르다.

* **`this` / `this()`:**
    * **`this.필드`**: 매개변수와 필드 이름이 같을 때 인스턴스 필드임을 명시한다.
    * **`this()`**: 클래스 내의 다른 생성자를 호출하여 코드 중복을 줄인다. (생성자 첫 줄에 위치 필수)
* **`super` / `super()`:**
    * **`super.필드`**: 부모 클래스의 멤버에 접근할 때 사용한다.
    * **`super()`**: 부모 클래스의 생성자를 호출한다. 자식 생성자의 첫 줄에서 부모 객체를 먼저 초기화해야 한다.

## 💻 실습 코드 분석 (Essential Snippets)

### ① Static 접근 제한 (`AClassTest.java`)
```java
public class AClassTest {
    int i = 7;           // 인스턴스 필드
    static int i2 = 7;   // 정적 필드

    public static void main(String[] args) {
        // System.out.println(i); // Error: static 영역에서 인스턴스 직접 접근 불가
        System.out.println(i2);   // OK: 메모리에 이미 존재함

        AClassTest test = new AClassTest();
        System.out.println(test.i); // OK: 객체 생성 후 참조 변수로 접근
    }
}
