## 💡 학습 내용 (Main Concepts)

### 1. Static 멤버 vs 인스턴스 멤버 (`AClassTest.java`)

* **Static (정적) 멤버:** 클래스가 메모리에 로드될 때 즉시 생성되며, 객체 생성 없이 클래스 이름으로 접근 가능하다.
* **인스턴스 멤버:** 반드시 `new` 연산자를 통해 객체가 생성되어야만 메모리(Heap)에 올라가며 사용할 수 있다.
* **주의사항:** `static` 메서드(예: main) 내부에서는 아직 메모리에 생성되지 않은 인스턴스 필드나 메서드에 직접 접근할 수 없다.

### 2. 메서드 호출 구조와 제어 흐름 (`BMethodTest2.java`)

* **Caller & Worker:** 메서드를 실행시키는 쪽(Caller)과 실행되는 쪽(Worker)으로 나뉜다.
* **제어권 이동:** 메서드가 호출되면 실행 흐름이 해당 메서드로 이동하며, 메서드 실행이 완료(종료)되어야 다시 호출한 곳으로 돌아온다.

### 3. 메서드의 입력과 출력 (`BMethodTest3.java`)
* **파라미터 (Parameter):** 메서드 실행에 필요한 데이터를 외부로부터 전달받는 매개체이다.
* **Return 문:** 메서드의 실행을 종료하고 결과값을 호출한 곳으로 돌려보낸다.
    * `void`: 반환할 데이터가 없을 때 사용한다.
    * `return` 이후에 작성된 코드는 실행되지 않으며 컴파일 에러가 발생한다.

### 4. 메서드 오버로딩 (Method Overloading) (`BMethodTest5.java`)

* **정의:** 하나의 클래스 내에 이름이 같은 메서드를 여러 개 선언하는 것이다.
* **성립 조건:** 메서드 이름이 같더라도 **파라미터의 개수나 데이터 타입**이 달라야 한다.
* **오해 금지:** 리턴 타입만 다르고 파라미터가 같은 경우는 오버로딩으로 인정되지 않는다.

## 💻 실습 코드 분석 (Code Insight)

### [AClassTest.java] - 메모리 로딩의 이해
```java
public class AClassTest {
    int i = 7; // 인스턴스 필드
    static int i2 = 7; // 정적 필드

    public static void main(String[] args) {
        // System.out.println(i); // Error: static 영역에서 인스턴스 영역 접근 불가
        System.out.println(i2); // OK: 메모리에 이미 존재함
        
        AClassTest test = new AClassTest();
        System.out.println(test.i); // OK: 인스턴스화 이후 접근 가능
    }
}

public void sum(int a, int b) {}
public void sum(int a, int b, int c) {} // 파라미터 개수가 다름 (OK)
public void sum(double a, double b) {}  // 데이터 타입이 다름 (OK)
// public int sum(int a, int b) {}      // 리턴 타입만 다른 것은 불가능 (Error)
```
