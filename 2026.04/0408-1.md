# 📅 TIL: 2026-04-08 (챕터 5. 참조 타입 - 배열)

> **Key Concepts:** `#Array`, `#Index`, `#Heap_Memory`, `#Reference_Variable`

배열은 같은 타입의 데이터를 연속된 공간에 나열하고, 각 데이터에 인덱스(Index)를 부여해 놓은 자료구조이다. 자바에서 배열은 객체로 취급되며 참조 타입에 속한다.

---

## 💡 학습 내용 (Main Concepts)

### 1. 배열의 특징
* **연속된 공간:** 데이터가 메모리상에 차례대로 배치된다.
* **고정된 길이:** 한번 생성된 배열은 실행 중에 길이를 늘리거나 줄일 수 없다. 
* **동일한 타입:** 같은 종류의 데이터만 저장 가능하다.

### 2. 배열의 선언과 생성
배열 변수를 선언하는 것과 실제 배열 객체를 생성하는 것은 별개의 과정이다.
* **선언:** `int[] scores;` 또는 `int scores[];`
* **생성 (new 연산자):** `scores = new int[5];` (5개의 정수를 저장할 공간 확보)
* **값 목록으로 생성:** `int[] scores = { 90, 80, 70 };` (선언과 동시에 값을 넣을 때만 가능)

### 3. 배열과 메모리 (Stack & Heap)
배열 변수는 **참조 변수**이다. 
* **Stack:** 배열 변수가 저장되며, 실제 배열이 위치한 **힙 영역의 주소값**을 가진다.
* **Heap:** 실제 데이터(인덱스 0, 1, 2...)가 저장되는 실제 배열 객체가 생성된다.
* 만약 `scores = null;`을 대입하면 스택의 연결이 끊기고, 주인을 잃은 힙 영역의 배열 객체는 **Garbage Collector**가 수거한다.

### 4. 배열의 인덱스와 길이
* **인덱스:** 0부터 시작한다. (마지막 인덱스는 `배열길이 - 1`)
* **length 필드:** `scores.length`를 통해 배열의 전체 길이를 읽을 수 있다. (수정 불가능한 읽기 전용)

## 📑 [핵심 암기 노트] 배열 사용 시 주의사항

| 구분 | 내용 | 비고 |
| :--- | :--- | :--- |
| **초기값** | `new`로 생성 시 숫자는 `0`, 객체는 `null`로 자동 초기화 | 중요! |
| **Index Error** | `ArrayIndexOutOfBoundsException` | 인덱스 범위를 벗어날 때 발생 |
| **길이 변경** | 불가 | 더 큰 배열을 만들고 복사(System.arraycopy)해야 함 |

## 💭 오늘의 요약 및 느낀 점
- **핵심 요약:** 배열은 대량의 데이터를 효율적으로 관리할 수 있는 가장 기초적인 도구이며, 인덱스를 활용한 빠른 접근이 큰 장점이다.
- **나의 생각:** 참조 타입의 원리를 배열에 대입해 보니 메모리 구조가 더 명확해졌다. 특히 `scores.length`가 단순한 숫자가 아니라 객체 내부의 속성이라는 점이 흥미로웠다. 내일은 다차원 배열과 향상된 for문을 더 깊게 파봐야겠다.
