# 📅 TIL: 2026-04-08 (클래스 멤버와 상속의 기초)

> **Key Concepts:** `#Static`, `#Constructor_Overloading`, `#Inheritance`, `#Override`

클래스가 메모리에 로딩되는 방식(Static)과 객체가 생성되는 다양한 방법, 그리고 코드의 재사용성을 극대화하는 상속을 학습했다.

---

## 💡 학습 내용 (Main Concepts)

### 1. Static (정적 멤버)
* **특징:** 객체 생성 없이 클래스 이름으로 바로 접근 가능한 공용 멤버이다.
* **Static 초기화 블록:** 클래스가 메모리에 처음 로드될 때 딱 한 번 실행되어 복잡한 정적 필드를 초기화한다.


### 2. 생성자 오버로딩 (Constructor Overloading)
* **개념:** 객체 생성 시 필요한 매개변수에 따라 여러 가지 생성자를 준비하는 기법이다.
* **효과:** 사용자가 원하는 재료(파라미터)에 맞춰 다양한 방식으로 객체를 탄생시킬 수 있다.

### 3. 상속 (Inheritance)
* **개념:** 부모 클래스의 필드와 메서드를 자식 클래스가 물려받는 것이다.
* **장점:** 코드의 재사용성이 높아지고 중복을 줄일 수 있다.
* **Override:** 부모에게 물려받은 메서드가 자식에게 맞지 않을 경우, 내용을 고쳐서 재정의할 수 있다.

---

## 📑 [핵심 암기 노트] Static vs Instance

| 구분 | Static 멤버 | Instance 멤버 |
| :--- | :--- | :--- |
| **메모리** | 메소드 영역 (공용) | 힙 영역 (개별) |
| **접근 방법** | 클래스명.멤버 | 참조변수.멤버 |
| **생성 시점** | 클래스 로드 시 | 객체 생성(`new`) 시 |
